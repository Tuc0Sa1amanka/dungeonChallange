# Dungeon Challenge Processor

## Описание

Программа обрабатывает внешние события, описывающие прохождение подземелья игроками.  
На вход подаются:
- Конфигурационный файл `config.json` (количество этажей, монстров на этаже, время открытия, длительность).
- Файл событий `events.txt` в формате `[HH:MM:SS] PlayerID EventID [ExtraParam]`.

На выходе:
- Лог обработанных событий (регистрация, вход, убийства, переходы, урон, восстановление, смерть, дисквалификация, невозможные действия).
- Финальный отчёт по каждому игроку: статус (`SUCCESS`/`FAIL`/`DISQUAL`), общее время в подземелье, среднее время зачистки обычного этажа, время убийства босса, здоровье.

## Основная логика

- **Регистрация** – только до закрытия подземелья. Повторная регистрация – `impossible move`.
- **Вход** – только в открытое подземелье и только один раз.
- **Убийство монстров** – возможно только на обычных этажах, пока не убиты все монстры этажа.
- **Переход на следующий этаж** – только после полной зачистки текущего этажа (все монстры убиты).
- **Переход на предыдущий этаж** – разрешён в любой момент (даже если на текущем этаже ещё есть монстры). При уходе с этажа, если на нём ещё есть монстры, накопленное время увеличивается на интервал от последнего входа до момента ухода.
- **Зачистка этажа** – фиксируется в момент убийства последнего монстра. Время, проведённое на этаже после этого, **не учитывается** (ни при расчёте среднего времени, ни при накоплении).
- **Босс** – доступен после полной зачистки всех обычных этажей. При входе выдаётся уведомление. Время убийства босса = разница между временем убийства и временем входа на его этаж. Покинуть этаж босса до его убийства нельзя.
- **Здоровье** – не может превышать 100. Восстановление выше 100 ограничивается 100.
- **Смерть** – при получении урона, который снижает здоровье до 0, игрок помечается как `finished`, дальнейшие события игнорируются.
- **Дисквалификация** – наступает при:
  - любом действии (кроме регистрации) незарегистрированного игрока,
  - регистрации после закрытия подземелья,
  - событии `9` (cannot continue).
- **Завершение испытания** – если время закрытия наступило, а игрок ещё активен, он принудительно завершается (`FAIL` или `SUCCESS` в зависимости от прогресса).

## Расчёт времени на этаже

Для каждого обычного этажа ведётся **суммарное время нахождения на этаже до зачистки**:
- При входе на этаж (первый раз или возврат) запоминается `lastEnter`.
- При уходе с этажа (переход на другой этаж, выход из подземелья, смерть, дисквалификация), если этаж ещё не очищен, к накопленному времени добавляется `текущее_время - lastEnter`.
- В момент убийства последнего монстра на этаже к накопленному времени добавляется последний интервал (если игрок в этот момент находится на этаже) – полученная сумма становится временем зачистки этажа.

**Среднее время зачистки** = сумма времён зачистки всех обычных этажей, делённая на их количество (выводится только для `SUCCESS`, иначе `00:00:00`).

**Общее время в подземелье** = `ExitTime - EntryTime` (если игрок вышел) или `CloseTime - EntryTime` (если время закрытия наступило раньше выхода).
